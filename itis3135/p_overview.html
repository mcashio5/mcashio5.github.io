Client Project – Part 1: Requirements & Design Proposal

Project Name

Dragonfly Home Market

1. Project Overview

1.1 Application Purpose

Dragonfly Home Market is a read-only online catalog for surplus furniture from Dragonfly Homes, a residential staging company. Its purpose is to showcase gently used staging pieces that are ready to be sold, reduce storage costs, and make it easier for buyers to browse inventory online. Dragonfly Home Market is a small, read-only online catalog for surplus furniture owned by Dragonfly Homes, a residential staging company. The purpose of the application is to showcase gently used staging pieces that are ready to be sold, reduce surplus inventory, and make it easier for potential buyers to see what is available without needing to visit a warehouse or active staging job.

1.2 Intended Users

  • Local homeowners and renters in the Charlotte area seeking discounted, high-quality furniture.
  • Real estate agents who need furniture quickly for listings.
  • Staging clients who want to purchase pieces used in their staged homes.
  • Dragonfly Homes staff who will share item and collection links with buyers.
  • Local homeowners and renters in the Charlotte area who want quality, gently used furniture at a discount.
  • Real estate agents who want to quickly furnish a listing with pieces from Dragonfly Homes.
  • Current and past staging clients who want to purchase items that were used in their staged homes.
  • Staff at Dragonfly Homes who will share links to specific pieces or collections with potential buyers.

1.3 Overview of Website Content

The site will contain:

  • Home page introducing Dragonfly Homes and the furniture sale concept.
  • Shop Inventory page listing available pieces with photos, descriptions, and pricing.
  • Styled Rooms Gallery showing staged rooms with notes about which items are for sale.
  • About page describing the business, its services, and background.
  • How to Buy / Contact page explaining the purchase process and providing an inquiry form.
  • A home page that introduces Dragonfly Homes, explains why furniture is being sold, and highlights featured categories.
  • A shop inventory page that lists individual furniture items with photos, descriptions, and prices, along with filters for room and category.
  • A styled rooms gallery that shows complete staged rooms and labels items that are currently available for purchase.
  • An “About Dragonfly Homes” page describing the business, services offered, and a short story of how the company started.
  • A “How to Buy / Contact” page with instructions for purchasing, pickup, and an inquiry form for specific pieces.

5.2 Shop Inventory (shop.html)

  • Name of the page: Shop Inventory
  • Purpose of the page: Display currently available furniture pieces with details so users can decide which items they want to inquire about.
  • Audience/Users of page: Primary customers who want to purchase furniture; Dragonfly Homes staff sharing item links.
  • Content of the page:
    • A grid or list of item cards with photo, name, short description, condition, dimensions, and price.
    • Filters for category (sofa, chair, table, decor), room (living room, bedroom, dining room, office), and price range.
    • “Interested” or “Add to Wishlist” buttons on each item.
  • Is this a page where you are asking users to enter data? Only through interactive controls such as filter selections and optional quantity selectors, but not through a full data-storing form.
  • Will those data fields need any validations? Filter selections will be validated on the client side to ensure appropriate ranges (for example, minimum price is not greater than maximum price).
  • Buttons, hyperlinks, drop downs:
    • Drop down lists for category and room filters.
    • jQuery UI slider for price range.
    • Buttons on each item: “View Details” or “Interested in this piece.”
    • Link at the bottom to the How to Buy / Contact page.
  • Actions that will happen on the page:
    • JavaScript filters the visible items instantly based on user selections, without reloading the page.
    • Clicking “Interested” for an item highlights it in a temporary wishlist panel (stored only in the current session).
    • Clicking “Contact about selected items” navigates to the Contact page and can pre-fill a message with the selected item names.
  • Special notes: Inventory data will be loaded via AJAX from a JSON file to make it easier to update stock without editing HTML.

  • Audience/Users of page: Serious buyers ready to contact Dragonfly Homes.

  • Special notes: The FAQ section below the form may use a jQuery UI accordion for collapsible questions and answers.

6. Dynamic Functionality on the Website

Filtering and Searching Inventory (Shop Inventory page)

  • JavaScript will filter visible item cards based on category, room, and price range, updating the list immediately when the user changes a filter.
  • A jQuery UI slider will be used for selecting the price range.
  • Similar interactivity can be seen on large furniture sites like IKEA or Wayfair, where users narrow down products without leaving the page.

AJAX Loading of Furniture Data (Shop Inventory page)

  • The furniture items will be stored in a JSON file.
  • When the Shop page loads, JavaScript will send an AJAX request to retrieve the JSON data and then dynamically create the item cards.
  • This allows the catalog to be updated by editing the JSON file instead of rewriting HTML.

Interactive Image Gallery / Lightbox (Styled Rooms Gallery page)

  • Clicking a gallery thumbnail will open the larger image in a modal overlay using jQuery or a lightbox plugin.
  • Users can move between images using previous and next controls within the modal.
  • Similar galleries exist on many portfolio and photography websites.

Wishlist Within the Session (Shop Inventory page)

  • When users click “Interested” on an item, JavaScript will add that item to a temporary wishlist displayed in a sidebar or panel.
  • The wishlist will exist only in memory for the current browser session and will not be stored in a database, respecting the course rule against CRUD functionality.

Form Validation and Confirmation (How to Buy / Contact page)

  • JavaScript will validate required fields, check email format, and display inline error messages.
  • On successful submission, the page will show a confirmation message (for example, “Thank you, we will contact you soon about your selected items.”).
  • This behavior is similar to contact or inquiry forms on many small business websites.

Optional jQuery UI Accordion for FAQs (How to Buy / Policies page)

  • Inventory filtering on Shop page: JavaScript/jQuery to filter items by category, room, and price using dropdowns and a jQuery UI slider.
  • AJAX-loaded catalog: Load item data from a JSON file so inventory can be updated without editing HTML.
  • Image gallery lightbox: jQuery-based gallery on the Styled Rooms page, with modal view and next/previous controls.
  • Wishlist within session: Temporary “Interested” list stored in memory for the current visit only.
  • Form validation: JavaScript validation on the How to Buy / Contact form for required fields and email format.
  • jQuery UI accordion: Optional FAQ accordion on the Policies/Contact page.
  • The FAQ section may use a jQuery UI accordion widget so that each question expands or collapses when clicked.
  • This keeps the page compact while still providing detail for users who want it.
