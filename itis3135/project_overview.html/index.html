Client Project – Part 1: Requirements & Design Proposal

Project Name

Dragonfly Home Market

1. Project Overview

1.1 Application Purpose

Dragonfly Home Market is a small, read-only online catalog for surplus furniture owned by Dragonfly Homes, a residential staging company. The purpose of the application is to showcase gently used staging pieces that are ready to be sold, reduce surplus inventory, and make it easier for potential buyers to see what is available without needing to visit a warehouse or active staging job.

1.2 Intended Users of the Website

  • Local homeowners and renters in the Charlotte area who want quality, gently used furniture at a discount.
  • Real estate agents who want to quickly furnish a listing with pieces from Dragonfly Homes.
  • Current and past staging clients who want to purchase items that were used in their staged homes.
  • Staff at Dragonfly Homes who will share links to specific pieces or collections with potential buyers.

1.3 Overview of the Content of the Website

The site will contain:

  • A home page that introduces Dragonfly Homes, explains why furniture is being sold, and highlights featured categories.
  • A shop inventory page that lists individual furniture items with photos, descriptions, and prices, along with filters for room and category.
  • A styled rooms gallery that shows complete staged rooms and labels items that are currently available for purchase.
  • An “About Dragonfly Homes” page describing the business, services offered, and a short story of how the company started.
  • A “How to Buy / Contact” page with instructions for purchasing, pickup, and an inquiry form for specific pieces.

5. Dynamic Functionality on the Website

Filtering and Searching Inventory (Shop Inventory page)

  • JavaScript will filter visible item cards based on category, room, and price range, updating the list immediately when the user changes a filter.
  • A jQuery UI slider will be used for selecting the price range.
  • Similar interactivity can be seen on large furniture sites like IKEA or Wayfair, where users narrow down products without leaving the page.

AJAX Loading of Furniture Data (Shop Inventory page)

  • The furniture items will be stored in a JSON file.
  • When the Shop page loads, JavaScript will send an AJAX request to retrieve the JSON data and then dynamically create the item cards.
  • This allows the catalog to be updated by editing the JSON file instead of rewriting HTML.

Interactive Image Gallery / Lightbox (Styled Rooms Gallery page)

  • Clicking a gallery thumbnail will open the larger image in a modal overlay using jQuery or a lightbox plugin.
  • Users can move between images using previous and next controls within the modal.
  • Similar galleries exist on many portfolio and photography websites.

Wishlist Within the Session (Shop Inventory page)

  • When users click “Interested” on an item, JavaScript will add that item to a temporary wishlist displayed in a sidebar or panel.
  • The wishlist will exist only in memory for the current browser session and will not be stored in a database, respecting the course rule against CRUD functionality.

Form Validation and Confirmation (How to Buy / Contact page)

  • JavaScript will validate required fields, check email format, and display inline error messages.
  • On successful submission, the page will show a confirmation message (for example, “Thank you, we will contact you soon about your selected items.”).
  • This behavior is similar to contact or inquiry forms on many small business websites.

Optional jQuery UI Accordion for FAQs (How to Buy / Policies page)

  • The FAQ section may use a jQuery UI accordion widget so that each question expands and collapses when clicked.
  • This keeps the page compact while still providing detail for users who want it.
